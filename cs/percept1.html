Perceptron 1: Linear Classification

Date: 07/28/2018

Classification of a linear line between two sets of data

The goal of this program is to linearly classify two sets of data, as shown above.

The Perceptron is a learning algorithm that utilizes two defined weights and a bias. What the program does is it first randomizes the weights, and then checks the set of data to see if it matches.
If everything matches, it stops! As the data is linearly classified correctly (even if not correct in terms of linear regression).
If there exists a point that is in the incorrect boundary, the scanning stops. And then, the coordinates are fed into an algorithm that modifies the weights (and the bias accordingly). This can be summarized as:

New_weight = Old_weight + learn_rate(target - output) * coordinate
New_bias = learn_rate(target - output)
Target is what we want the data point to be. In this case red wants to be above, so its target is 1e Output is 0 if the line were to be above the point. For blue it is vice versa.
The learn_rate is an arbitrary number used to dictate how quickly the perceptron 'learns'.

Some challenges that came about while making this includes:
1. The window was scaled to (-300, 300) instead of (0, 1): This made the algorithm extremely wacky if not out of control.
2. The line would spiral out of control multiple times. Turns out once the line turns negative, the 'above' for the line actually becomes below the line! This was fixed by absolute valuing the scanning function.
Overall, pretty fun.
